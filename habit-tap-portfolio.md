# Habit Tap ポートフォリオ資料

## アプリ概要

Habit Tapは、毎日の習慣を手軽に記録できるWebアプリです。

ユーザーは「水を飲む」「勉強する」「運動する」などの習慣を登録し、その日の達成状況をワンタップで記録できます。リマインド通知、週間カレンダー、スキップ、メモ、並び替え機能を備え、無理なく習慣を継続できることを目指しました。

ログイン不要で使える静的Webアプリとして作成しており、データはブラウザのlocalStorageに保存されます。

## 主な機能

- 習慣の追加、編集、削除
- 今日の達成チェック
- スキップ記録
- 習慣ごとのメモ保存
- 習慣の上下並び替え
- 過去7日分の週間カレンダー表示
- 達成数、スキップ数、未達成数、達成率の表示
- 連続達成日数の表示
- リマインド時刻の設定
- ブラウザ通知
- ブラウザ内でのデータ保存
- スマホ、PC対応のレスポンシブデザイン

## 工夫した点

習慣管理アプリは多機能になりすぎると継続しにくいため、基本操作を1画面で完結できるようにしました。

達成、スキップ、メモ、並び替えなどの操作を習慣カード内にまとめることで、ユーザーが迷わず操作できるUIを意識しました。

スキップ機能では、体調不良や予定変更などで実行できない日があっても、連続記録が途切れないように設計しました。完璧に続けることよりも、無理なく継続することを重視しています。

また、GitHub Pagesで公開しやすいように、外部ライブラリやサーバーを使わず、HTML/CSS/JavaScriptのみで実装しました。

## 開発の流れ

1. 企画
   - 幅広い人が使いやすいWebアプリとして、習慣チェックアプリを選定
   - 毎日使いやすく、入力の手間が少ないアプリを目標に設定

2. 要件定義
   - 習慣の登録、達成チェック、リマインド通知、データ保存をMVPとして定義
   - ログインやクラウド同期は初期開発の対象外に設定

3. 設計
   - 静的Webアプリとして構成
   - データ保存にはlocalStorageを使用
   - GitHub Pagesで公開できる構成に整理

4. 開発
   - 習慣追加、編集、削除、達成チェックを実装
   - リマインド通知とサマリー表示を追加
   - 追加アップデートとして、週間カレンダー、並び替え、メモ、スキップ機能を実装

5. テスト
   - JavaScriptの構文チェックを実施
   - ブラウザで習慣追加、達成チェック、スキップ、メモ保存、並び替え、再読み込み後の保存維持を確認
   - GitHub Pagesで公開し、反映状況を確認

## 制作で学んだこと

静的Webアプリでも、localStorageやブラウザ通知を活用することで、実用的なアプリを作成できることを学びました。

また、GitHub Pagesで公開する際には、ファイル配置やキャッシュ対策が重要であることも理解しました。特に、CSSやJavaScriptの変更がすぐ反映されない場合に、読み込みURLへバージョンを付けることでキャッシュを回避できる点は実践的な学びでした。

開発の進め方としては、最初にMVPを小さく作り、その後に週間カレンダーやメモ機能などを段階的に追加することで、アジャイル的に改善を重ねる流れを体験できました。

今回の制作を通じて、機能を増やすだけでなく、ユーザーが続けやすい設計にすることの大切さを学びました。
